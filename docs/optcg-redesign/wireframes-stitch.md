## Developer Implementation Notes

### How to use these files

The HTML files in `stitch-screens/` are the authoritative visual reference for all UI components in this redesign. The developer agent must:

1. Read each HTML file to extract visual structure, spacing, class patterns, and layout decisions.
2. Map the Stitch-generated classes and inline styles to the project's Tailwind token system (defined in `frontend/tailwind.config.js` and `src/index.css`).
3. Do not copy Stitch HTML verbatim — adapt it to the existing React component structure while preserving the visual design intent.
4. Any deviation from the Stitch screens must be documented in "Open Questions / Risks" in the implementation.

### Token mapping reminder

The Stitch screens use generic CSS values. Map them to semantic tokens:

| Stitch value | Project semantic token | Tailwind class |
|--------------|----------------------|----------------|
| `#FAFAFA` bg | `--color-page` (light) | `bg-page` |
| `#0B132B` bg | `--color-page` (dark) | `bg-page` |
| `#FFFFFF` card bg | `--color-card` (light) | `bg-card` |
| `rgba(28,37,65,0.85)` card bg | `--color-card` (dark) | `bg-card` |
| `#1A1A2E` text | `--color-text-primary` (light) | `text-primary` |
| `#FFFFFF` text | `--color-text-primary` (dark) | `text-primary` |
| `rgba(0,0,0,0.55)` text | `--color-text-secondary` (light) | `text-secondary` |
